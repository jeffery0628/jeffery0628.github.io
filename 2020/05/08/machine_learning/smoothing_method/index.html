平滑方法

更新于 2020-09-20 分类于 技术 ， 机器学习 阅读次数： 评论：
本文字数： 4.3k 阅读时长 ≈ 4 分钟

简介

在一个概率模型中，也就是 p(e)在 event space E 下的概率分布，模型很可能会用最大似然估计(MLE)：
\[ P_{M L E}=\frac{c(x)}{\sum_{e} c(e)} \]
然而，由于并没有足够的数据，很多事件 \(x\) 并没有在训练数据中出现，也就是 \(c(x)=0\)，\(P_{MLE}=0\)这是有问题的，没有在训练数据中出现的数据，并不代表不会在测试数据中出现，如果没有考虑到数据稀疏性，模型就显得太简单了。

Data sparsity 是 smoothing 的最大原因。Chen & Goodman 在1998 年提到过，几乎所有数据稀疏的场景下，smoothing 都可以帮助提高 performance，而数据稀疏性几乎是所有统计模型都会遇到的问题。而如果你有足够多的训练数据，所有的 parameters 都可以在没有 smoothing 的情况下被准确的估计，那么你总是可以扩展模型，如原来是 bigram，没有数据稀疏，完全可以扩展到 trigram 来提高 performance，如果还没有出现稀疏，就再往高层推，当 parameters 越来越多的时候，数据稀疏再次成为了问题，这时候，用合适的smoothing_method可以得到更准确的模型。实际上，无论有多少的数据，平滑几乎总是可以以很小的代价来提高 performance。

smoothing_method

add-one smoothing

(也叫laplace smoothing),以bigram为例：

MLE estimate：
\[ P_{M L E}\left(w_{i} | w_{i-1}\right)=\frac{c\left(w_{i-1} w_{i}\right)}{c\left(w_{i-1}\right)} \]
Add-one estimate:
\[ P_{A d d-1}\left(w_{i} | w_{i-1}\right)=\frac{c\left(w_{i-1} w_{i}\right)+1}{c\left(w_{i-1}\right)+V} \]
其中，\(V\)表示词典大小。

假设语料库为：

  1. john read moby dick
  2. mary read a different book
  3. she read a book by cher

那么，john read a book 这个句子的概率为：

\[ \begin{aligned} p(\text { john read a book })&=p(\text { john } |<s>) p(\text { read } | \text { john }) p(\text { a } | \text { read }) p(\text { book } | a ) p(<e> | \text {book}) \\ &=\frac{c(<s>,john)}{\sum_{w} c(<s>,w)} \frac{c(john,read)}{\sum_{w} c(john,w)} \frac{c(read,a)}{\sum_{w} c(read,w)} \frac{c(a,book)}{\sum_{w} c(a,w)} \frac{c(book,<e>)}{\sum_{w} c(book,w)}\\ &= \frac{1}{3} * \frac{1}{1} * \frac{2}{3} * \frac{1}{2} * \frac{1}{2} \\ &\approx 0.006 \end{aligned} \]

而，cher read a book 这个句子出现的概率是：

\[ \begin{aligned} p(\text { cher read a book })&=p(\text { cher } |<s>) p(\text { read } | \text { cher }) p(\text { a } | \text { read }) p(\text { book } | a ) p(<e> | \text {book}) \\ &=\frac{c(<s>,cher)}{\sum_{w} c(<s>,w)} \frac{c(cher,read)}{\sum_{w} c(cher,w)} \frac{c(read,a)}{\sum_{w} c(read,w)} \frac{c(a,book)}{\sum_{w} c(a,w)} \frac{c(book,<e>)}{\sum_{w} c(book,w)}\\ &= \frac{0}{3} * \frac{0}{1} * \frac{2}{3} * \frac{1}{2} * \frac{1}{2} \\ &\approx 0 \end{aligned} \]
如果使用add-one smoothing：
\[ \begin{aligned} p(\text { john read a book })&=p(\text { john } |<s>) p(\text { read } | \text { john }) p(\text { a } | \text { read }) p(\text { book } | a ) p(<e> | \text {book}) \\ &= \frac{1+1}{3+11} * \frac{1+1}{1+11} * \frac{2+1}{3+11} * \frac{1+1}{2+11} * \frac{1+1}{2+11} \\ &\approx 0.0001 \end{aligned} \]

\[ \begin{aligned} p(\text { cher read a book })&=p(\text { cher } |<s>) p(\text { read } | \text { cher }) p(\text { a } | \text { read }) p(\text { book } | a ) p(<e> | \text {book}) \\ &= \frac{0+1}{3+11} * \frac{0+1}{1+11} * \frac{2+1}{3+11} * \frac{1+1}{2+11} * \frac{1+1}{2+11} \\ &\approx 0.00003 \end{aligned} \]

加 1 平滑通常情况下是一种很糟糕的算法，与其他smoothing_method相比显得非常差，可以把加 1 平滑用在其他任务中，如文本分类，或者非零计数没那么多的情况下。

Additive smoothing

对加 1 平滑的改进就是把 1 改成 \(\delta\)，且 \(0<\delta\)
\[ P_{A d d-\delta}\left(w_{i} | w_{i-1}\right)=\frac{c\left(w_{i-1} w_{i}\right)+\delta}{c\left(w_{i-1}\right)+\delta V} \]
可以把\(\delta\)看作是一个可以调节的超参数。

Good-Turing smoothing

基本思想: 用观察计数较高的 N-gram 数量来重新估计概率量大小，并把它指派给那些具有零计数或较低计数的 N-gram.

举个例子：假设你在钓鱼，然后抓到了 18 条鱼，种类如下：10条鲤鱼, 3条黑鱼, 2条刀鱼, 1条鲨鱼, 1条草鱼, 1条鳗鱼，那么

  1. 下一个钓到的鱼是鲨鱼的概率是多少？

    1/18

  2. 下一条与是新鱼种（之前没有出现过）的概率是多少？

    3/18(没有出现过的按出现过一次算)

  3. 下一条抓到的鱼是鲨鱼的概率是多少？

    首先肯定的是概率小于1/18 --> good turing

在good turing 下,对参数的估计分两种情况：

  1. 没有出现过的：\(P_{GT}=\frac{N_1}{N}\)
  2. 出现过的：\(P_{GT}=\frac{(c+1)N_{c+1}}{N_c*N}\)

其中，\(N_c\)表示出现c次的单词个数,\(N\)代表样本总数。
比如：

sam i am i am sam i do not eat

sam：2次

i：3次

am：2次

do：1次

not：1次

eat：1次

有：\(N_3=1,N_2=2,N_1=3\)

对于第3个问题：\(P_{GT}=\frac{(1+1)*1}{3*18}=\frac{1}{27}\).

缺点

由公式：\(P_{GT}=\frac{(c+1)N_{c+1}}{N_c*N}\)在计算当前项\(c\) 的时候，需要计算到\(c+1\)的\(N_{c+1}\),若\(N_{c+1}=0\)，则会导致概率为0.

解决方法：可以利用机器学习中回归算法对\(N_1,N_2,\ldots,N_c,N_{c+1}\)进行拟合，进而得出\(N_{c+1}\)的值。

Interpolation

基本思想：在计算Trigram概率的同时，考虑Unigram，Bigram，Trigram出现的次数。

举个例子：有

C(in the kitchen)=0,C(the kitchen)=3,C(kitchen)=4,C(arboretum)=0.在计算p(kitchen|in the)和p(arboretum|in the)的时候，从语料计算得出：\(p(kitchen|in,the)=p(arboretum|in,the)=0\),但是从经验上来讲\(p(kitchen|in,the)>p(arboretum|in,the)\)，因为kitchen要比arboretum常见的多。要实现这个，我们就希望把 bigram 和 unigram 结合起来，interpolate 就是这样一种方法。用线性差值把不同阶的 N-gram 结合起来，这里结合了 trigram，bigram 和 unigram。用 lambda 进行加权
\[ \begin{aligned} \mathrm{p}\left(\mathrm{w}_{\mathrm{n}} | \mathrm{w}_{\mathrm{n}-1}, \mathrm{w}_{\mathrm{n}-2}\right)=& \lambda_{1} \mathrm{p}\left(\mathrm{w}_{\mathrm{n}} | \mathrm{w}_{\mathrm{n}-1}, \mathrm{w}_{\mathrm{n}-2}\right) \\ &+\lambda_{2} \mathrm{p}\left(\mathrm{w}_{\mathrm{n}} | \mathrm{w}_{\mathrm{n}-1}\right) \\ &+\lambda_{3} \mathrm{p}\left(\mathrm{w}_{\mathrm{n}}\right)\\ \sum_i\lambda_i=1 \end{aligned} \]
